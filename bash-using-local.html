bash using local

Using the bash builtin command local is like using declare. The only difference is that variables declared using local inherit a local binding attribute for functions, meaning that the name referencing the local variable only exists inside the function declared up until subsequent calls to local using the same name reference.

bash -c "help local"
local: local [option] name[=value] ...
    Define local variables.

    Create a local variable called NAME, and give it VALUE.  OPTION can
    be any option accepted by `declare'.

    Local variables can only be used within a function; they are visible
    only to the function where they are defined and its children.

    Exit Status:
    Returns success unless an invalid option is supplied, a variable
    assignment error occurs, or the shell is not executing a function.

Note the help page for declare provides a more detailed treatment for variable attribute options.

how to declare a local integer

If you are asking "how do I declare a local integer?" There are 5 ways. Pick your poison. Personally, I like what is behind door number 2. That way you can pretend that declare and local don't do exactly the same thing even though they do.

the first way: use local

local -i myint

the second way: use local and declare

local myint
declare -i myint

the third way: use subshell

( declare -i myint )

the forth way: use subshell with source i

( . source-of-script-containing-myint )

the fifth way: use subshell with source ii

bash source-of-script-containing-myint

style guide

How I use local

Use local to map positional parameters to namerefs

If you are not declaring one of these

_() { # anonymous function like construct that you may use
  ... # do whatever you want because this function doesn't even have a name
}

or are using another convention to pass arguments to a function, then I recommend mapping positional parameters to namerefs before anything happens within the function as follows.

fun() { { local a ; a="${1}" ; local b ; b=${@:2} ; } # signature
  ... 
}

All variables are local unless expected

Unless a variable is expected to be inherited from the global scope or a caller function declare any variable use by a function local.

fun() { # signature
  ## declare local variables here
  local int
  local arr
  #etc...
}

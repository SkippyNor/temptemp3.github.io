apache tips

You can use some tips for Apache HTTP Server to aid in troubleshooting issues, host setup ,and optimizing performance.

1. do not trust documentation

In most cases documentation provided by a software vender serves as a source of information that can be trusted; however, there are certain situations in which you would not want to trust documentation.

do not trust non-English documentation with "may be outdated" notice

A while back you noticed the "may be outdated" notice in some non-English apache documentation, and didn't think much of it — at first.

Translation of Japanese documentation

Apache Module mod_headers

Apache HTTP Server Version 2.4 (in Japanese)

この日本語訳はすでに古くなっている 可能性があります。 最近更新された内容を見るには英語版をご覧下さい。

https://httpd.apache.org/docs/current/ja/mod/mod_headers.html

Translation:

Apache HTTP Server Version 2.2

Not good.

How to interpret Available Languages

Example)

Apache Module mod_mime

Apache HTTP Server Version 2.4 (in English)

Available Languages: en | fr | ja

https://httpd.apache.org/docs/current/en/mod/mod_mime.html

Under Available Languages: en (English); fr (French); and ja (Japanese); are listed. It is the opinion of the author that ja should not be listed, because it is most likely outdated.

In that respect, do not trust non-English documentation with "may be outdated" notice. In the above case fr is okay, if you understand French; if not, stick to English. Furthermore, even if you understand Japanese, don't read anything with a "may be outdated" notice.

ftp recursively download files

Somewhere down the road you may be tasked with continuing a project in a staging environment with nothing but files and access via old-fashion ftp. As they say, once you go git you never want to git backwards to unfettered data transfer through ftp. At the only logical choice is to move out.

You could download the files on your machine but that would not only be lame but blew up dust. Find a host that you have ssh access to and recursively download files through ftp using wget.

wget -r ftp://${host} --user="${user}" --password="${passwd}"

When it's done a day or so later, pack up and download the payload.

tar -czf ${host}.tar.gz ${host}

Wrapping up

Suppose the goal is to complete the task to one go. You are going to need the following two commands.

ssh() { command ssh ... ; }

scp() { command ssh ... ; }

There are effectively three hosts:

  1. Your machine or origin of script
  2. The host with ssh access
  3. The host with only ftp holding all your files

On host-1, we make a payload script. Suppose that we clipboy make:function script-1 a command for this. Call this script-1.

script-1 has subcommands. One subcommand we will call execute and another cleanup. Let's assume they work because whoever wrote them did a good job.

scp script-1 # place payload

ssh "script-1 run" # execute payload on host-2 interacting with host-3

scp script-1-payload ./ # copy to host-1 from host-2

ssh 'script-1 cleanup' # cleanup

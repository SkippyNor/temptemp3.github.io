list of GitHub Pages tips

1. make sure your content is utf-8

Whether you like it or not, the http HEAD response for all of your GitHub Pages will conveniently let the user agent, in most cases your browser, that your content charset is utf-8, which it may not correct.

Here is an example HEAD response.

curl -Ik https://temptemp3.github.io/bash-script-coding-style-guide -s
HTTP/2 200
date: Sat, 05 Aug 2017 15:46:07 GMT
server: GitHub.com
content-type: text/html; charset=utf-8
strict-transport-security: max-age=31557600
last-modified: Sat, 05 Aug 2017 15:43:15 GMT
access-control-allow-origin: *
expires: Sat, 05 Aug 2017 15:55:03 GMT
cache-control: max-age=600
x-github-request-id: 7FDA:1C4EF:2E8056:31F5D1:5985E7FF
accept-ranges: bytes
via: 1.1 varnish
age: 64
x-served-by: cache-nrt6121-NRT
x-cache: HIT
x-cache-hits: 1

vary: Accept-Encoding
x-fastly-request-id: 59420d208e085246c39ebb9b5f36c5cd30912c5e
content-length: 7874

Even if you've devised an experimental scheme to set the charset dynamically, the charset indicated indicated in the HEAD response may supercied the markup of your document depending on browser behavior.

Make sure your content is utf-8 to avoid unwanted scrambled characters in your content as you may have already seen in css w3 modals.

2. deploy subtree updates to master branch

Deploy only files required by your GitHubs Pages page, i.e. html files, css, ect. To accomplish this you may want to push required files to a subtree to be merged to your master branch. I have laid out the procedure to deploy subtree updates to master branch in another post.

3. use a static site generator

Using a static site generator in GitHubs Pages is a must. Not only does it save you time not having to generate html files for each document you create individually, it allows you to focus more on creating content. In addition, due to the fact that pages are not hand-coded, you are able to introduce dynamic-like components into your pages such as: structured data; rankings; and other components often seen on dynamic sites.

There is an endless array of generators out there such as: jekyll (ruby), hugo (go), and hexo (javascript), to name a few.

This blog is generated using sh2-u2, a static site generator written in bash.

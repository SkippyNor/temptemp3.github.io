bash bin rm argument list to long

You were trying to remove files from a directory with rm and stumbled upon the '-bash: /bin/rm: Argument list too long' error message (21 Sep 2017)

rm -rvf path-to-files/*
-bash: /bin/rm: Argument list too long

(oops)

What happened

Before passing a list of file names to rm, 'path-to-files/*' is expanded to all files and directories contained in 'path-to-files', which we will call k.

The rm command has a set limit on the number of arguments that is is able to handle in a single command line invocation that we will call j.

In calling rm with the expandable 'path-to-files/*' argument, k > j, triggering an error response.

Workaround

Use find and xargs in combination with rm

This allows you to limit the amount to arguments supplied to rm by calling rm k times.

I.e.

find path-to-files -maxdepth 1 | xarg rm -rvf
